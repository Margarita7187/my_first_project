1. Link просто осуществляет навигацию, NavLink дополнительно добавляет класс active или стиль к ссылке когда маршрут активный. NavLink полезен для навигационных меню.

2. Через хук useParams() для динамических параметров (:id), useSearchParams() для query-параметров, useLocation() для полной информации о location.

3. Программная навигация - изменение маршрута из кода без клика по ссылке. Реализуется через хук useNavigate(): navigate('/path') или navigate(-1) для назад.

4. Через state в navigate('/path', {state: {data}}) или через search-параметры. Данные доступны через useLocation().state.

5. Routes оборачивает коллекцию Route компонентов и выбирает первый подходящий маршрут для отображения. Заменяет старый Switch из v5.

6. Защищённый маршрут создаётся как компонент-обёртка, который проверяет аутентификацию и либо рендерит дочерний компонент, либо перенаправляет на логин.

7. useNavigate - хук для программной навигации внутри компонентов, redirect - компонент для перенаправления в JSX. redirect рендерится и сразу выполняет навигацию.
