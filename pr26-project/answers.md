# Ответы на вопросы для самоконтроля

## 1. Какие основные проблемы legacy PHP-кода вы выявили?
- Смешение PHP и HTML в одном файле
- Уязвимости безопасности (SQL-инъекции, XSS)
- Отсутствие архитектуры MVC
- Прямые SQL-запросы без ORM
- Нет обработки ошибок

## 2. В чем преимущества использования фреймворка like Laravel?
- Структурированная архитектура MVC
- Встроенные механизмы безопасности
- Автоматизация типовых задач
- Eloquent ORM для работы с БД
- Шаблонизатор Blade

## 3. Как работает MVC-архитектура в Laravel?
- **Model** (app/Models) - работа с данными через Eloquent
- **View** (resources/views) - Blade шаблоны для отображения
- **Controller** (app/Http/Controllers) - обработка логики приложения

## 4. Что такое Composer и для чего он используется?
Composer - менеджер зависимостей PHP. Используется для:
- Установки пакетов и фреймворков
- Автозагрузки классов
- Управления версиями зависимостей

## 5. Какие возможности предоставляет Blade-шаблонизатор?
- Наследование шаблонов (@extends)
- Вставка переменных {{ $var }}
- Условные операторы @if, @foreach
- Включение подшаблонов @include
- Экранирование данных

## 6. Как организована маршрутизация в Laravel?
Маршруты определяются в routes/web.php:
```php
Route::get('/url', function() { return view; });
Route::get('/user/{id}', [Controller::class, 'method']);
```

## 7. Какие меры безопасности предоставляет Laravel по сравнению с чистым PHP?
- Защита от SQL-инъекций через Eloquent
- Встроенная CSRF-защита
- Экранирование в Blade
- Валидация данных
- Шифрование и хеширование

## 8. Как происходит передача данных в шаблоны?
Через второй параметр view():
```php
return view('template', ['name' => 'John', 'data' => $array]);
```
В шаблоне данные доступны как переменные: `{{ $name }}`
