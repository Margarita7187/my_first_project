1. Viewport meta tag - это тег в HTML который сообщает браузеру как обрабатывать размеры и масштабирование страницы на мобильных устройствах. Он важен потому что без него мобильные браузеры будут отображать страницу как на десктопе с последующим уменьшением что делает текст нечитаемым и нарушает адаптивность.

2. В работе использовался комбинированный подход: Fluid для гибких контейнеров и изображений которые плавно изменяют размеры, Adaptive для дискретных изменений через медиа-запросы, и Responsive как общая стратегия сочетающая оба метода для обеспечения корректного отображения на всех устройствах.

3. Mobile First подход означает что стили сначала пишутся для мобильных устройств а затем через медиа-запросы min-width добавляются стили для более широких экранов. Его преимущества включают лучшую производительность так как мобильные устройства загружают только нужные стили, более простую поддержку кода и приоритет контента над дизайном.

4. Для адаптивной верстки наиболее предпочтительны проценты для ширины контейнеров, rem для размеров шрифта и отступов так как это обеспечивает доступность, vw/vh для элементов которые должны занимать часть viewport и em для компонентов где нужна относительность к родительскому шрифту. Эти единицы обеспечивают гибкость и масштабируемость.

5. Функция clamp() позволяет задать минимальное предпочтительное и максимальное значение для свойства что идеально для типографики например clamp(1rem 2.5vw 2rem) создает шрифт который масштабируется между 1rem и 2rem в зависимости от ширины viewport но никогда не выходит за эти границы обеспечивая читаемость на всех устройствах.

6. Основные брейкпоинты 768px для планшетов и 1024px для десктопов были выбраны на основе распространенных разрешений устройств и особенностей контента когда текущая верстка начинала выглядеть неоптимально. Эти значения соответствуют типичным точкам перелома в адаптивном дизайне.

7. Медиа-запросы @media позволяют применять CSS-стили только при выполнении определенных условий например при конкретной ширине экрана. Пример синтаксиса: @media (min-width: 768px) { .container { max-width: 720px; } } где стили применяются только когда ширина viewport составляет 768px и больше.

8. max-width устанавливает максимальную ширину при которой применяются стили и используется в Desktop First подходе min-width устанавливает минимальную ширину и используется в Mobile First. Max-width применяется для экранов меньше указанного значения min-width для экранов больше указанного.

9. Изображения делаются адаптивными через свойство max-width: 100% которое предотвращает выход изображения за границы контейнера и height: auto которое сохраняет пропорции. Дополнительно можно использовать srcset для загрузки разных версий изображений в зависимости от разрешения экрана.

10. Для тестирования адаптивного дизайна в браузерах используются инструменты разработчика особенно режим адаптивного дизайна который позволяет эмулировать различные устройства панель Elements для проверки применяемых стилей и Console для отладки JavaScript включая функциональность мобильного меню.
